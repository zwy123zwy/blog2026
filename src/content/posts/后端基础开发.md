# 后端基础开发知识框架

## 一、HTTP 与网络基础

### 1.1 HTTP 协议

- **请求与响应**：请求行、请求头、请求体；状态行、响应头、响应体
- **HTTP 方法**：GET、POST、PUT、PATCH、DELETE、HEAD、OPTIONS
- **状态码**：2xx（成功）、3xx（重定向）、4xx（客户端错误）、5xx（服务器错误）
- **HTTP/1.1 vs HTTP/2**：多路复用、头部压缩、服务器推送
- **HTTPS**：TLS/SSL、证书、加密流程

### 1.2 RESTful API 设计

- **REST 原则**：资源、HTTP 方法、无状态、统一接口
- **URL 设计**：资源命名、层级关系、版本控制
- **请求/响应格式**：JSON、Content-Type、Accept
- **认证与授权**：Token、JWT、OAuth 2.0、API Key
- **API 文档**：Swagger/OpenAPI、Postman

### 1.3 常见问题

- **幂等性**：GET、PUT、DELETE 的幂等性；POST 如何保证幂等
- **安全性**：SQL 注入、XSS、CSRF、参数校验、输入过滤
- **性能**：压缩、缓存、CDN、连接池

## 二、数据库

### 2.1 关系型数据库（SQL）

- **SQL 基础**：SELECT、INSERT、UPDATE、DELETE、JOIN（INNER、LEFT、RIGHT、FULL）
- **索引**：B+ 树、哈希索引、聚簇索引、非聚簇索引、索引优化
- **事务**：ACID（原子性、一致性、隔离性、持久性）
- **隔离级别**：READ UNCOMMITTED、READ COMMITTED、REPEATABLE READ、SERIALIZABLE
- **锁**：行锁、表锁、乐观锁、悲观锁、死锁
- **范式**：第一范式到第三范式、反范式化

### 2.2 非关系型数据库（NoSQL）

- **类型**：
  - **文档型**：MongoDB（BSON、集合、文档）
  - **键值型**：Redis（字符串、哈希、列表、集合、有序集合）
  - **列式**：Cassandra、HBase
  - **图数据库**：Neo4j
- **选型考量**：数据结构、一致性、性能、扩展性

### 2.3 数据库设计

- **表设计**：字段类型、主键、外键、约束
- **关系**：一对一、一对多、多对多
- **查询优化**：EXPLAIN、慢查询、索引优化、分页优化
- **分库分表**：垂直拆分、水平拆分、分片策略、路由

## 三、缓存

### 3.1 缓存策略

- **缓存位置**：浏览器缓存、CDN、反向代理、应用缓存、数据库缓存
- **缓存更新**：Cache Aside、Read/Write Through、Write Behind
- **缓存失效**：TTL、LRU、LFU、FIFO
- **缓存穿透**：空值缓存、布隆过滤器
- **缓存击穿**：互斥锁、热点数据预热
- **缓存雪崩**：过期时间随机化、多级缓存

### 3.2 Redis

- **数据结构**：String、Hash、List、Set、Sorted Set、Bitmaps、HyperLogLog
- **持久化**：RDB、AOF、混合持久化
- **主从复制**：主从、哨兵、集群模式
- **应用场景**：缓存、计数器、分布式锁、消息队列、会话存储

---

## 四、消息队列

### 4.1 消息队列基础

- **作用**：解耦、异步、削峰、可靠性
- **消息模型**：点对点、发布/订阅
- **消息可靠性**：持久化、确认机制、事务消息
- **消息顺序**：分区、顺序消费

### 4.2 常见消息队列

- **RabbitMQ**：AMQP、Exchange、Queue、Binding、消息确认
- **Kafka**：Topic、Partition、Consumer Group、Offset、高吞吐
- **RocketMQ**：顺序消息、事务消息、延迟消息
- **选型**：吞吐量、延迟、可靠性、功能特性

## 五、微服务架构

### 5.1 微服务基础

- **概念**：服务拆分、独立部署、服务治理
- **优势**：技术栈灵活、独立扩展、故障隔离
- **挑战**：服务间通信、数据一致性、分布式事务、服务发现

### 5.2 服务通信

- **同步通信**：HTTP/REST、gRPC、服务间调用
- **异步通信**：消息队列、事件驱动
- **服务发现**：Eureka、Consul、Nacos、DNS
- **负载均衡**：轮询、加权轮询、最少连接、一致性哈希

### 5.3 服务治理

- **API 网关**：路由、认证、限流、熔断、监控
- **配置中心**：集中配置、动态更新、版本管理
- **链路追踪**：分布式追踪、日志聚合、性能分析
- **服务监控**：指标收集、告警、健康检查

## 六、分布式系统

### 6.1 分布式基础

- **CAP 定理**：一致性（Consistency）、可用性（Availability）、分区容错性（Partition tolerance）
- **BASE 理论**：基本可用、软状态、最终一致性
- **分布式事务**：2PC、3PC、TCC、Saga、本地消息表、最大努力通知

### 6.2 分布式锁

- **实现方式**：Redis SETNX、Zookeeper、数据库锁
- **问题**：死锁、锁超时、可重入、公平锁
- **Redlock**：Redis 分布式锁算法

### 6.3 分布式 ID

- **生成方式**：UUID、数据库自增、Redis 自增、Snowflake、Leaf
- **要求**：全局唯一、趋势递增、高性能、高可用

---

## 七、系统设计

### 7.1 设计原则

- **可扩展性**：水平扩展 vs 垂直扩展、无状态设计
- **高可用**：冗余、故障转移、降级、熔断
- **性能优化**：缓存、CDN、数据库优化、异步处理
- **安全性**：认证授权、数据加密、防攻击

### 7.2 常见系统设计题

- **短链接系统**：URL 生成、存储、重定向、统计
- **秒杀系统**：限流、库存扣减、防刷、降级
- **聊天系统**：消息存储、实时推送、在线状态
- **搜索引擎**：爬虫、索引、检索、排序
- **推荐系统**：特征工程、召回、排序、冷启动

---

## 八、安全

### 8.1 常见攻击与防护

- **SQL 注入**：参数化查询、输入验证
- **XSS（跨站脚本）**：输入过滤、输出转义、CSP
- **CSRF（跨站请求伪造）**：Token、SameSite Cookie
- **DDoS**：限流、CDN、WAF
- **敏感数据**：加密存储、传输加密、脱敏

### 8.2 认证与授权

- **认证方式**：Session、Token、JWT、OAuth 2.0、SSO
- **密码安全**：哈希（bcrypt、argon2）、加盐、密码策略
- **权限控制**：RBAC（角色访问控制）、ABAC（属性访问控制）

---

## 九、性能优化

### 9.1 应用层优化

- **代码优化**：算法优化、减少数据库查询、批量操作
- **并发处理**：线程池、连接池、异步处理
- **缓存策略**：多级缓存、缓存预热、缓存更新

### 9.2 数据库优化

- **索引优化**：合理创建索引、避免索引失效
- **查询优化**：避免 SELECT *、LIMIT、分页优化
- **连接池**：数据库连接池、连接数配置

### 9.3 系统优化

- **负载均衡**：Nginx、LVS、硬件负载均衡
- **CDN**：静态资源加速、边缘缓存
- **监控与调优**：APM、日志分析、性能测试
