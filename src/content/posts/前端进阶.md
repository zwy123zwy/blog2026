# Js错误

## JavaScript 错误类型总览

| 错误类型           | 触发场景                | 示例代码                           | 解决方案                 |
| :----------------- | :---------------------- | :--------------------------------- | :----------------------- |
| **SyntaxError**    | 语法错误                | `if (a) {`                         | 修复代码语法             |
| **ReferenceError** | 访问未声明变量          | `console.log(undeclaredVar);`      | 声明变量或检查变量作用域 |
| **TypeError**      | 类型不匹配操作          | `null.func();`                     | 添加类型检查             |
| **RangeError**     | 数值超出有效范围        | `new Array(-1);`                   | 验证输入值范围           |
| **URIError**       | URI处理函数使用无效参数 | `decodeURIComponent('%');`         | 验证URI参数              |
| **EvalError**      | `eval()` 函数使用不当   | `new eval();`                      | 避免使用eval             |
| **AggregateError** | 多个错误组合            | `Promise.any([rejectedPromises]);` | 处理多个错误             |
| **自定义错误**     | 开发者主动抛出          | `throw new CustomError('msg');`    | 实现特定错误处理逻辑     |

### 1. SyntaxError（语法错误）

**触发时机**：代码解析阶段  
**常见场景**：

- 缺少括号、引号或花括号
- 错误使用保留字
- 无效的表达式

```javascript
// 示例1: 缺少括号
if (a {  // Uncaught SyntaxError: Unexpected token '{'

// 示例2: 无效赋值
const 123var = "test";  // Uncaught SyntaxError: Invalid or unexpected token
```

### 2. ReferenceError（引用错误）

**触发时机**：运行时  
**常见场景**：

- 访问未声明的变量
- 访问块作用域变量（let/const）的TDZ（暂时性死区）
- 模块导入错误

```javascript
// 示例1: 未声明变量
console.log(undeclaredVar);  // Uncaught ReferenceError

// 示例2: TDZ访问
{
  console.log(x);  // Uncaught ReferenceError
  let x = 5;
}
```

### 3. TypeError（类型错误）

**触发时机**：操作不符合变量类型  
**常见场景**：

- 调用非函数值
- 访问null/undefined的属性
- 尝试修改常量值

```javascript
// 示例1: 调用非函数
const num = 42;
num();  // Uncaught TypeError: num is not a function

// 示例2: 访问null属性
const obj = null;
console.log(obj.property);  // Uncaught TypeError: Cannot read property
```

### 4. RangeError（范围错误）

**触发时机**：参数超出有效范围  
**常见场景**：

- 无效数组长度
- 数字方法参数越界
- 递归深度过大

```javascript
// 示例1: 无效数组长度
new Array(-1);  // Uncaught RangeError: Invalid array length

// 示例2: 数字精度超限
(123.456).toFixed(101);  // Uncaught RangeError: toFixed() digits argument must be between 0 and 100
```

### 5. URIError（URI错误）

**触发时机**：URI处理函数参数无效  
**仅影响**：`encodeURI()`, `decodeURI()`, `encodeURIComponent()`, `decodeURIComponent()`

```javascript
// 示例: 无效URI解码
decodeURIComponent('%');  // Uncaught URIError: URI malformed
```

### 6. EvalError（eval错误）

**触发时机**：`eval()` 函数使用不当  
**注意**：现代 JavaScript 中很少见，保留用于兼容性

```js
// 历史用法 (现代JS引擎不再抛出)
try {
  throw new EvalError("Eval error");
} catch (e) {
  console.log(e instanceof EvalError);  // true
}
```

### 7. AggregateError（聚合错误）

**触发时机**：多个错误需要同时报告  
**常见于**：`Promise.any()` 全部拒绝时

```js
Promise.any([
  Promise.reject(new Error("Error 1")),
  Promise.reject(new Error("Error 2"))
]).catch(e => {
  console.log(e instanceof AggregateError);  // true
  console.log(e.errors.length);  // 2
});
```

#### WebIDL 与 ecma-262 中的错误类型

- **ECMAScript 异常**（[ECMAScript exceptions](https://tc39.es/ecma262/#sec-native-error-types-used-in-this-standard)）对应 **WebIDL 简单异常**：脚本运行时发生的错误会创建 Error 对象并抛出。除通用 `Error` 外，ECMAScript 2015 还定义了：
  - **EvalError**：eval 错误
  - **RangeError**：范围错误
  - **ReferenceError**：引用错误
  - **TypeError**：类型错误
  - **URIError**：URI 错误
  - **SyntaxError**：语法错误（WebIDL 中故意省略，由 ES 解析器使用）
  - **Error**：通用错误（WebIDL 中故意省略，供开发者使用）

- **DOMException**（[DOM 规范](https://dom.spec.whatwg.org/#exception-domexception)）：DOM 规范定义的错误类型集，兼容旧版 DOMError，常见包括：
  - **IndexSizeError**：索引不在允许范围
  - **HierarchyRequestError**：节点树层次结构不正确
  - **WrongDocumentError**：对象属于错误文档
  - **InvalidCharacterError**：字符串包含无效字符
  - **NoModificationAllowedError**：对象不可修改
  - **NotFoundError**：对象未找到
  - **NotSupportedError**：不支持的操作
  - **InvalidStateError**：对象处于无效状态
  - **SyntaxError**：字符串与预期模式不匹配
  - **InvalidModificationError**：对象不能以该方式修改
  - **NamespaceError**：在 XML 命名空间中不允许
  - **InvalidAccessError**：对象不支持该操作或参数
  - **TypeMismatchError**：对象类型与预期不匹配
  - **SecurityError**：操作不安全
  - **NetworkError**：网络错误
  - **AbortError**：操作被中止
  - **URLMismatchError**：URL 不匹配
  - **QuotaExceededError**：超出配额
  - **TimeoutError**：操作超时
  - **InvalidNodeTypeError**：节点或祖先节点类型不正确
  - **DataCloneError**：对象无法克隆

## 前端错误异常的捕获方式

| 方式 | 说明 |
|------|------|
| **try-catch** | ES 提供的语法，仅能捕获**同步代码**中的异常，无法捕获回调、setTimeout、Promise 内抛错 |
| **window.onerror**（DOM0） | 可捕获运行时错误；资源加载错误需在**元素**上绑定 onerror（如 img、script、link） |
| **window.addEventListener('error', cb, true)**（DOM2） | 捕获阶段监听，可捕获资源加载失败（不冒泡到 window 的 error） |
| **window.addEventListener('unhandledrejection', cb)**（DOM4） | 全局捕获未处理的 Promise rejection |
| **Promise.then().catch(cb)** | 在链式调用中显式捕获 Promise 错误 |
| **覆写 XMLHttpRequest / fetch** | 在请求封装层统一处理 HTTP 错误与网络异常 |

### try-catch-finally

将可能抛出异常的代码放在 `try` 中，用 `catch` 接收错误对象并处理，`finally` 无论如何都会执行。

```javascript
try {
  throw new Error("会有错误的代码块");
} catch (e) {
  report(e);
} finally {
  console.log("finally");
}
```

### onerror 事件

#### window.onerror

当 JavaScript **运行时错误**（含语法错误）发生时，会触发 `ErrorEvent` 并执行 `window.onerror`。  
注意：**语法错误**会导致当前脚本块执行失败，若语法错误出现在错误监控代码所在脚本块之前，可能无法被监控到。

```javascript
/**
 * @param {string} message 错误信息
 * @param {string} source  发生错误的脚本 URL
 * @param {number} lineno  行号
 * @param {number} colno   列号
 * @param {object} error   Error 对象
 */
function err(message, source, lineno, colno, error) {
  // 上报 message, source, lineno, colno, error
}
window.onerror = err;
```

#### element.onerror

当**资源**（如 `<img>`、`<script>`、`<link>`）加载失败时，该元素会触发 `Event` 的 error 事件，执行元素上的 `onerror`。  
注意：此类 error **不会冒泡到 window**，但可通过 `window.addEventListener('error', cb, true)` 在捕获阶段统一处理。

```javascript
element.onerror = function (event) { /* 处理资源加载失败 */ };
```

#### window.addEventListener('error', cb, true)

在**捕获阶段**监听 error，可同时捕获**运行时错误**和**资源加载错误**。可通过 `event` 类型或属性区分（如 `event.target !== window` 表示资源错误）。

```javascript
window.addEventListener(
  "error",
  function (e) {
    var eventType = [].toString.call(e);
    if (eventType === "[object Event]") {
      // 可进一步根据 e.target 判断为资源加载错误
      report(e);
    }
  },
  true
);
```

#### window.addEventListener('unhandledrejection', cb)

用于全局捕获**未处理的 Promise rejection**。

```javascript
window.addEventListener("unhandledrejection", function (event) {
  // 处理未捕获的 rejection，如 event.reason
  report(event.reason);
  event.preventDefault(); // 阻止控制台默认报错
});
```

### Promise.then().catch(cb).finally()

在 Promise 链上显式使用 `.catch()` 捕获错误；在不支持 `unhandledrejection` 的环境可配合 polyfill 使用。

```javascript
new Promise(function (resolve, reject) {
  throw "Uncaught Exception!";
}).catch(function (e) {
  console.log(e); // Uncaught Exception!
});
```

### 覆写 XMLHttpRequest 与 fetch

在封装层统一监听请求失败（如 status 非 2xx、error、abort），便于上报与重试。

```javascript
// 覆写 XMLHttpRequest
if (!window.XMLHttpRequest) return;
var xmlhttp = window.XMLHttpRequest;
var _oldSend = xmlhttp.prototype.send;
var _handleEvent = function (event) {
  if (event && event.currentTarget && event.currentTarget.status !== 200) {
    report(event);
  }
};
xmlhttp.prototype.send = function () {
  if (this["addEventListener"]) {
    this["addEventListener"]("error", _handleEvent);
    this["addEventListener"]("load", _handleEvent);
    this["addEventListener"]("abort", _handleEvent);
    this["addEventListener"]("close", _handleEvent);
  } else {
    var _oldStateChange = this["onreadystatechange"];
    this["onreadystatechange"] = function (event) {
      if (this.readyState === 4) {
        _handleEvent(event);
      }
      _oldStateChange && _oldStateChange.apply(this, arguments);
    };
  }
  return _oldSend.apply(this, arguments);
};

// 覆写 fetch
if (!window.fetch) return;
var _oldFetch = window.fetch;
window.fetch = function () {
  return _oldFetch
    .apply(this, arguments)
    .then(function (res) {
      if (!res.ok) {
        report(res);
      }
      return res;
    })
    .catch(function (error) {
      report(error);
    });
};
```
